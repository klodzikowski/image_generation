Variational Autoencoder (VAE) Foundation

A VAE learns to compress images into a tiny latent space, then reconstruct them. Instead of memorising images pixel-by-pixel, it learns a smooth, continuous space where similar images are nearby. This is the foundation for almost all modern generative models.

Key insight: The VAE doesn't store images — it learns a compressed representation. The latent space is continuous: you can smoothly interpolate between a "3" and an "8" and get plausible digits in between. This is why VAEs (and their latent spaces) are the backbone of Stable Diffusion, DALL-E, and other modern generators — diffusion happens in the latent space, not on raw pixels.

How does a VAE work?

  • Encoder: Compresses a 28x28 image (784 pixels) down to just 2 numbers (the latent code z). It outputs a distribution (mean + variance), not a single point.
  • Reparameterisation trick: Sample from that distribution: z = μ + σ × ε (where ε ~ N(0,1)). This makes the model trainable via backpropagation.
  • Decoder: Takes the 2 numbers and reconstructs the full 28x28 image.
  • Loss = Reconstruction + KL divergence. Reconstruction ensures the output looks like the input. KL divergence ensures the latent space is smooth and well-organised (close to a standard normal).

Why MNIST?

MNIST (handwritten digits) is the classic VAE example because the data is simple enough that a 2D latent space can capture it. You can literally see the latent space: digits cluster by identity, and the space between clusters produces smooth transitions. With real images, the same principle applies — but the latent space has 4–256 dimensions instead of 2.

Diffusion: Learning to Denoise Most Mature

A diffusion model starts from pure random noise and iteratively removes it, guided by the text prompt. Each step makes the image a little cleaner. Watch it happen:

Step 0 / 50 — Pure noise
Key insight: The model doesn't "draw" the image. It starts with noise and removes it step by step. Early steps establish layout & colour; later steps add texture & fine detail. This is why you can stop early and get a blurry but recognisable image.

What's happening at each step?

The neural network (a U-Net or Transformer) looks at the current noisy image + the text prompt and predicts "what noise is present." That predicted noise is subtracted, producing a slightly cleaner image. Repeat 10–50 times.

Autoregressive: Treating Images as Language Fastest-improving

Autoregressive models tokenise the image (like words) and predict tokens sequentially. But "sequential" doesn't mean pixel-by-pixel. Toggle between the two approaches:

Select a mode above, then press Generate
Raster scan (DALL-E 1, 2021): Tokens are predicted one by one, left-to-right, top-to-bottom — like typing a letter. This is slow (1,024 sequential steps) and produces poor spatial coherence because the model can't "see" what's below or to the right. This approach is obsolete.
Next-Scale / VAR (NeurIPS 2024 Best Paper): The model generates a tiny 1×1 thumbnail first, then 2×2, then 4×4… up to full resolution. At each scale, all tokens are generated in parallel. Only ~10 autoregressive steps total (scale→scale), not 1,024. It's like a painter sketching the rough composition first, then adding detail. 20× faster than diffusion.

Hybrid / Transfusion Dominant approach (2025+)

The approach behind GPT-4o and Google's Nano Banana. Text is generated autoregressively (token by token). But when the model decides to produce an image, it generates all image patches simultaneously via diffusion.

Press Generate to see the full pipeline
What's "autoregressive" here? The decision to create an image and the surrounding text are autoregressive: "Draw" → "a" → "cat" → BOI → [image] → EOI → "Here's your cat…". But the image itself is generated via parallel diffusion on all patches at once — like a Polaroid developing. The AR framework is the scaffolding; diffusion is the brush.

The key takeaway: "Autoregressive image generation" does not mean the image appears pixel by pixel like a typewriter. Only the obsolete raster-scan approach (DALL-E 1) worked that way. Modern AR models generate coarse-to-fine (VAR) or use diffusion for the actual pixels (Transfusion/GPT-4o).
